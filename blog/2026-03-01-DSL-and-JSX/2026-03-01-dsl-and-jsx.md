도큐사우루스라는 프로그램으로 회사에서 기술 문서를 만들고 있다. 더 구체적으로 말하면 웹 매뉴얼을 작성할 때 사용 중인데, 마이크로소프트의 Word와는 사용법이 많이 다르다.

그 덕분에 생각지도 못했던 HTML, CSS, 자바스크립트 등 웹 개발에 사용되는 언어를 공부하고 있다. 여기서 더 나아가 리액트(React)라는 이름의 라이브러리도 익혀야 했다. 도큐사우루스가 리액트를 기반으로 만들어진 프로그램이기 때문이었다.

여러 언어를 동시에 살펴보다보니 낯선 문법이 나왔을 때, 자바스크립트에 해당하는지 리액트에 해당하는지 헷갈릴 때가 있었다. 클로드에게 리액트에서만 지원하는 문법을 물어봤더니 가장 먼저 JSX를 알려주었다.

다음은 클로드가 JSX에 대해 설명한 내용이다.

> JSX는 HTML처럼 생긴 문법을 JavaScript 안에서 쓸 수 있게 해줍니다. 순수 JavaScript에는 없는 문법으로, React가 내부적으로 변환해서 처리합니다.

```
// JSX — React에서만 가능
function Hello() {
  return <h1>안녕하세요!</h1>;
}

// 순수 JavaScript로 변환하면
function Hello() {
  return React.createElement('h1', null, '안녕하세요!');
}
```

오늘은 JSX를 본격적으로 학습하기에 앞서, DSL을 정리해보았다.

### DSL의 정의

- DSL(Domain Specific Language)
  - 도메인 특화 언어
  - 특정 문제 영역에 최적화
  - 자바스크립트나 파이썬처럼 범용적으로 사용되는 다목적 프로그래밍 언어(GPL)와 구분
  - 외부 DSL와 내부 DSL로 구분

### 외부 DSL

- 독자적인 문법과 파서를 가지며, 별개의 파일에서 독립적으로 실행되는 언어
- 예: YAML, SQL, CSS
- 각자의 영역에서 표현력이 뛰어남
- 정해진 문법 규칙을 엄격하게 지켜야 함
- 익히기까지 별도의 학습 곡선을 가짐

### 내부 DSL

- HTML은 그 자체로 외부 DSL이며 브라우저가 직접 해석할 수 있음
  - 자바스크립트로 DOM을 동적으로 조작하려면 별도의 스크립트에서 명령형 코드를 작성해야 함
  - HTML과 자바스크립트의 경계는 개발 과정을 번거롭게 만듦
- 내부 DSL은 범용적인 프로그래밍 언어의 문법과 기능을 확장
  - 새로운 언어처럼 보이게 만들며 특정 기능을 수행하는 역할을 하는 언어
- JSX(JavaScript XML)
  - HTML의 태그와 유사하게 생긴 코드를 사용
  - 자바스크립트 파일 내부에서 UI 구조를 작성하는 용도로 만듦
  - 자바스크립트에서 파생된 내부 DSL
  - UI의 구조와 동작 로직을 하나의 자바스크립트 파일 안에서 선언적으로 관리할 수 있음
  - HTML과 자바스크립트를 자연스럽게 넘나들 수 있다는 점이 JSX를 강력하게 만드는 핵심
